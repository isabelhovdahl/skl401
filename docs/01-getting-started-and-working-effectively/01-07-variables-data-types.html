Variables and basic data types

We introduce variables as named containers for data and explore Python’s basic data types: integers, floats, strings, and booleans.

# Assigning a value to a variable
x = 5
print(x)

# Use variables in calculations
y = 10
print(x + y)

# Overwrite a variable
x = 5
x = 100
print(x)

# Good variable names
monthly_revenue = 5000
gdp_per_capita_2007 = 82500

# Avoid these names
# MonthlyRevenue = 50000   # CamelCase — works but not conventional in Python
# x = 50000                # Too vague for real code
# print = 50000            # Will overwrite print() function
# 2nd_quarter = 50000      # Invalid — cannot start with a number

# Integer (whole numbers)
quantity = 7

# Float (decimal numbers)
price = 99.75

# String (text, enclosed in single or double quotes)
product = "coffee"

# Boolean (True or False - note the capital letters)
in_stock = True

print(type(quantity))    # <class 'int'>
print(type(price))       # <class 'float'>
print(type(product))     # <class 'str'>
print(type(in_stock))    # <class 'bool'>

# Convert integer to float
print(float(quantity))

# Convert float to integer
print(int(price))

# Round a float
print(round(price))    # round to nearest integer
print(round(price, 1)) # round to nearest one decimal

# A number stored as a string 
val_as_text = str(quantity)
print(val_as_text)
print(type(val_as_text))  

# Convert number stored as string
print(int("42"))

# TypeError - Cannot convert a word to a number
# print(int("coffee"))

Exercises

Exercise 1.7.1

Without running the code, predict the value that will be printed on the last line. Then run the code to check your answer. What does this tell you about how variable assignment works in Python?

x = 10
x = x + 5
x = x * 2
print(x)
Solution 
30
Step by step: x starts as 10, then becomes 10 + 5 = 15, then becomes 15 * 2 = 30. Each assignment overwrites the previous value without any warning. This means you need to be careful with variable names in longer scripts — if you reuse a name by accident, the old value is gone and Python will not tell you.

Exercise 1.7.2

Some of the variable names below are invalid or go against Python conventions. Identify which ones are problematic, explain what is wrong with each, and suggest a corrected name.

2025_sales = 15000
monthly revenue = 8000
FirstName = "Julia"
int = 42
gdp_per_capita = 55000
Solution 
# Invalid — variable names cannot start with a number
sales_2025 = 15000

# Invalid — variable names cannot contain spaces
monthly_revenue = 8000

# Valid but not conventional — Python uses snake_case, not CamelCase
first_name = "Julia"

# Valid but dangerous — this overwrites Python's built-in int() function,
# meaning int() will no longer work for the rest of the session
n_int = 42

# No problem — this is a good variable name
gdp_per_capita = 55000
The first two will cause an immediate SyntaxError. The third works but goes against Python conventions — use snake_case for variable names. The fourth is the most subtle: it runs without error but silently breaks int() for the rest of your session. Always avoid using the names of built-in Python functions as variable names.

Exercise 1.7.4

Before running any code, look at each variable below and write down what type you think it is. Then check your answers using type(). Were any of them surprising?

a = 5.0
b = 4 / 2
c = "4"
d = True
e = int(4.9)
f = round(4.9)

Once you have checked the types, answer the following:

  1. What is the difference between e and f? What does this tell you about the difference between int() and round()?

  2. What do you think will happen if you try float("2") and int("2.0")? Predict the result, then run it and check.

Solution 
a = 5.0
b = 4 / 2
c = "4"
d = True
e = int(4.9)
f = round(4.9)

print(type(a))
print(type(b))
print(type(c))
print(type(d))
print(type(e))
print(type(f))
<class 'float'>
<class 'float'>
<class 'str'>
<class 'bool'>
<class 'int'>
<class 'int'>

A common surprise is b: even though 4 / 2 divides evenly and produces a whole number result, the output is always a float in Python. Division with / always returns a float regardless of the operands. If you need an integer result, use integer division // instead: 4 // 2 returns 2 as an int.

  1. e = int(4.9) gives 4 — int() simply truncates the decimal part. f = round(4.9) gives 5 — round() rounds to the nearest integer. They give different results whenever the decimal part is 0.5 or above, so it is important to choose the right one for your purpose.

  2. float("2.0") works fine and returns 2.0. But int("2.0") raises a ValueError:

ValueError: invalid literal for int() with base 10: '2.0'
Even though "2.0" looks like a whole number, int() cannot convert a string that contains a decimal point — it only accepts strings that look exactly like integers, such as "2". The safe approach when converting a string that may contain decimals is to go via float first: int(float("2.0")) works and returns 2. This situation comes up often when importing data from files where whole numbers have been stored with a trailing decimal point.

Exercise 1.7.5

You are given the following variables, but two of them have been stored as the wrong type and will cause problems. Identify which ones they are, explain why they are problematic, and fix them so that the final calculation runs correctly and prints a sensible result.

product   = "coffee"
price     = "12.50"
quantity  = 3.0
discount  = 0.10

total = price * quantity * (1 - discount)
print(f"Total cost for {product}: {total}")
Solution

Running the code as-is raises a TypeError:

TypeError: can't multiply sequence by non-int of type 'float'

Two variables have the wrong type:

  • price = "12.50" is stored as a string. It should be a float since it represents a numeric value that we need to do arithmetic with.
  • quantity = 3.0 is stored as a float. This is not strictly wrong — the calculation will work — but a quantity of items is a whole number and should be an integer.
product   = "coffee"
price     = float("12.50")   # or simply: price = 12.50
quantity  = int(3.0)         # or simply: quantity = 3
discount  = 0.10

total = price * quantity * (1 - discount)
print(f"Total cost for {product}: {total}")
Total cost for coffee: 33.75
In practice, type mismatches like this occur often when importing data from files — numeric columns are sometimes read in as strings and need to be converted before you can use them in calculations.
