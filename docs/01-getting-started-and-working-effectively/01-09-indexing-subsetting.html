Indexing and subsetting

We cover how to extract individual elements and slices from lists and strings using Python’s zero-based indexing system.

products = ["coffee", "tea", "juice", "sandwich", "cookie"]

# Get the first element (position 0)
print(products[0])    # coffee

# Get the third element (position 2)
print(products[2])    # juice

# Get the last element
print(products[len(products)-1])    # cookie

# Negative indexing
print(products[-1])   # cookie (last element)
print(products[-2])   # sandwich (second to last)

product = {
    "name": "coffee",
    "price": 29.90,
    "in_stock": True
}

# Cannot index a dict
# print(product[0])

# Extract value associated with key
print(product["name"])    # coffee

# Elements at positions 1, 2, 3 (stop=4 is excluded)
print(products[1:4])    # ['tea', 'juice', 'sandwich']

# From the beginning up to (but not including) position 3
print(products[:3])     # ['coffee', 'tea', 'juice']

# From position 2 to the end
print(products[2:])     # ['juice', 'sandwich', 'cookie']

# Slicing a string
country = "Norway"
print(country[0])      # N
print(country[-1])     # y
print(country[0:3])    # Nor
print(country[3:])     # way

# Modify element in list at index 0
products[0] = "apples"
print(products)

Unlike lists, which raise an IndexError when you go out of bounds, dictionaries raise a KeyError when you try to access a key that does not exist. Both are signals that you are asking for something that is not there.

Exercise 1.9.4

Apply slicing to the string below to extract the following substrings and print each result: 1. The first three characters. 2. The last four characters using negative indexing. 3. The word "Python" (characters 7 through 12).

text = "Hello, Python!"
Solution 
text = "Hello, Python!"

# 1. First three characters
print(text[:3])

# 2. Last four characters
print(text[-4:])

# 3. The word "Python"
print(text[7:13])
Hel
hon!
Python
Remember that the stop index in a slice is excluded, so to include the character at position 12 you must use 13 as the stop index.

Exercise 1.9.5

The code below contains three common indexing mistakes. Identify and fix each one, and explain what was wrong.

products = ["coffee", "tea", "juice", "sandwich", "cookie"]

# 1. Retrieve the first element
print(products[1])

# 2. Retrieve the last element
print(products[5])

# 3. Retrieve the first three elements
print(products[0:2])
Solution 
products = ["coffee", "tea", "juice", "sandwich", "cookie"]

# 1. First element is at position 0, not 1
print(products[0])

# 2. The list has 5 elements (positions 0–4); position 5 does not exist
print(products[-1])   # or products[4]

# 3. The slice [0:2] only returns positions 0 and 1 — stop index is excluded
print(products[0:3])
coffee
cookie
['coffee', 'tea', 'juice']
All three mistakes are variations of the same off-by-one confusion that comes from zero-based indexing. A list of 5 elements has valid positions 0 through 4 — there is no position 5. And a slice [0:2] returns 2 elements (positions 0 and 1), not 3. These errors are extremely common and worth recognising early.
