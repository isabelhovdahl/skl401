Line charts

We cover line charts for showing how a numeric variable changes over time.

import matplotlib.pyplot as plt
import pandas as pd
import seaborn as sns

gapminder = pd.read_csv("data/gapminder.csv")

# ── Preparing the data ───────────────────────────────────────

# Average life expectancy per continent per year
continent_life = gapminder.groupby(["continent", "year"])["lifeExp"].mean().reset_index()

continent_life["continent"].unique()
continent_life["year"].nunique()
continent_life.head()

# ── A single line ────────────────────────────────────────────

africa = continent_life[continent_life["continent"] == "Africa"]

fig, ax = plt.subplots(figsize=(9, 5))

ax.plot(africa["year"], africa["lifeExp"])

ax.set_xlabel("Year")
ax.set_ylabel("Life expectancy (years)")
ax.set_title("Life expectancy in Africa over time")

plt.show()
plt.close()

# ── Adding all continents with a loop ────────────────────────

# Loop over unique continents, filter and plot each one
# label= is used by ax.legend() to populate the legend
fig, ax = plt.subplots(figsize=(9, 5))

for continent in continent_life["continent"].unique():
    group = continent_life[continent_life["continent"] == continent]
    ax.plot(group["year"], group["lifeExp"], label=continent)

ax.set_xlabel("Year")
ax.set_ylabel("Life expectancy (years)")
ax.set_title("Life expectancy over time by continent")
ax.legend(title="Continent")

plt.show()
plt.close()

# ── Line charts with dates on the x-axis ────────────────────

# The dowjones dataset: monthly closing price of the Dow Jones Industrial Average
dowjones = sns.load_dataset("dowjones")
dowjones.shape
dowjones.info()
dowjones.head()

# Convert Date from string to datetime
dowjones["Date"] = pd.to_datetime(dowjones["Date"])

fig, ax = plt.subplots(figsize=(10, 5))

ax.plot(dowjones["Date"], dowjones["Price"])

ax.set_xlabel("Date")
ax.set_ylabel("Price")
ax.set_title("Dow Jones Industrial Average")

plt.show()
plt.close()

# ── What happens when dates are strings ──────────────────────

# Without datetime conversion, tick labels are squashed raw text
dowjones["Date_str"] = dowjones["Date"].astype("str")

fig, ax = plt.subplots(figsize=(10, 5))

ax.plot(dowjones["Date_str"], dowjones["Price"])

plt.show()
plt.close()

# ── What happens when data is not sorted ─────────────────────

# matplotlib draws lines in row order — unsorted data produces zigzag lines
dowjones_shuffled = dowjones.sort_values("Price")

fig, ax = plt.subplots(figsize=(10, 5))

ax.plot(dowjones_shuffled["Date"], dowjones_shuffled["Price"])

plt.show()
plt.close()

Exercise 4.3.2

Practice the loop-over-groups pattern on the healthexp dataset using a different grouping variable than the video — individual countries rather than continents.

import pandas as pd
import matplotlib.pyplot as plt

healthexp = pd.read_csv("data/healthexp.csv")
countries = ["Germany", "Japan", "USA", "Great Britain"]
  1. Filter healthexp to just the four countries listed above.
  2. Create a Figure and Axes with figsize=(9, 5).
  3. Use a for loop over countries to plot each country’s life expectancy over time as its own line, using label= to set the country name.
  4. Add axis labels, a title, and call ax.legend() to display the labels.
  5. Which country has the highest life expectancy at the end of the period? Which had the lowest at the start?
Solution 
import pandas as pd
import matplotlib.pyplot as plt

healthexp = pd.read_csv("data/healthexp.csv")
countries = ["Germany", "Japan", "USA", "Great Britain"]

subset = healthexp[healthexp["Country"].isin(countries)]

fig, ax = plt.subplots(figsize=(9, 5))

for country in countries:
    group = subset[subset["Country"] == country]
    ax.plot(group["Year"], group["Life_Expectancy"], label=country)

ax.set_xlabel("Year")
ax.set_ylabel("Life expectancy (years)")
ax.set_title("Life expectancy over time by country")
ax.legend(title="Country")

plt.tight_layout()
plt.show()
  1. Japan both starts and ends the period with the highest life expectancy. The USA has the lowest life expectancy at the end of the period among these four countries, despite the highest spending.

Exercise 4.3.3

The file below contains quarterly data US GDP. Load it and see what happens when you plot a date column that has not been converted to datetime.

import pandas as pd
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/isabelhovdahl/skl401/refs/heads/master/data/FRED/GDP.csv"
gdp = pd.read_csv(url)
print(gdp.dtypes)
gdp.head()
  1. Without converting the DATE column, create a line chart of GDP over DATE. Use figsize=(12, 5). What does the x-axis look like? Is it readable?
  2. Now convert DATE to datetime using pd.to_datetime() and create the same chart again. How does the x-axis change?
  3. In your own words, explain why matplotlib is able to space and format the x-axis sensibly in the second chart but not the first.
Solution 
import pandas as pd
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/isabelhovdahl/skl401/refs/heads/master/data/FRED/GDP.csv"
gdp = pd.read_csv(url)

# 1. Plot with DATE as a string
fig, ax = plt.subplots(figsize=(12, 5))
ax.plot(gdp["DATE"], gdp["GDP"])
ax.set_title("GDP over time (DATE as string)")
plt.show()
# 2. Convert to datetime and replot
gdp["DATE"] = pd.to_datetime(gdp["DATE"])

fig, ax = plt.subplots(figsize=(12, 5))
ax.plot(gdp["DATE"], gdp["GDP"])
ax.set_xlabel("Year")
ax.set_ylabel("GDP (billions USD)")
ax.set_title("US GDP over time")
plt.tight_layout()
plt.show()

  1. With DATE as a string, matplotlib treats every unique date as a separate category on the x-axis and tries to print every single one as a text label. With over 300 quarters in the data, the labels overlap into an unreadable smear.

  2. Once DATE is a proper datetime, matplotlib understands that the x-axis represents time, not arbitrary categories. It can then choose sensible tick positions (e.g. one label every few years) and format them cleanly, because it knows how to compare, order, and space actual dates — something it cannot do with plain text.

Exercise 4.3.4

The DataFrame below contains a small time series that has been accidentally shuffled out of chronological order.

import pandas as pd
import matplotlib.pyplot as plt

sales = pd.DataFrame({
    "date":    pd.to_datetime([
        "2023-03-01", "2023-01-01", "2023-05-01", "2023-02-01",
        "2023-04-01", "2023-06-01"
    ]),
    "revenue": [42, 38, 51, 40, 47, 55]
})
sales["date"] = pd.to_datetime(sales["date"])
  1. Plot revenue against date exactly as the data is currently ordered, using figsize=(8, 5). What does the line look like? Is this what you would expect from steadily increasing monthly revenue?
  2. Sort sales by date and plot again. How does the chart change?
  3. In your own words, explain why matplotlib produced a “zigzag” in step 1 even though date was already a proper datetime column.
Solution 
import pandas as pd
import matplotlib.pyplot as plt

sales = pd.DataFrame({
    "date":    pd.to_datetime([
        "2023-03-01", "2023-01-01", "2023-05-01", "2023-02-01",
        "2023-04-01", "2023-06-01"
    ]),
    "revenue": [42, 38, 51, 40, 47, 55]
})
sales["date"] = pd.to_datetime(sales["date"])

# 1. Plot without sorting
fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(sales["date"], sales["revenue"])
ax.set_title("Revenue (unsorted)")
plt.show()
# 2. Sort and replot
sales_sorted = sales.sort_values("date")

fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(sales_sorted["date"], sales_sorted["revenue"])
ax.set_title("Revenue (sorted)")
plt.show()
  1. ax.plot() always draws a line connecting points in the order the rows appear in the DataFrame — it does not sort by the x-values for you, even when those x-values are proper dates. Since the rows in sales were not in chronological order, the line jumps back and forth across the x-axis rather than tracing a smooth path from January to June. Having a correct datetime dtype fixes how the axis is labeled, but it does not fix the plotting order — that always depends on row order, which is why sorting is a separate and necessary step.

Exercise 4.3.5

This exercise combines data preparation, datetime conversion, and sorting into one realistic end-to-end task using US monthly macroeconomic data.

import pandas as pd
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/isabelhovdahl/skl401/refs/heads/master/data/FRED/FRED_monthly_2010.csv"
fred = pd.read_csv(url)
print(fred.dtypes)
fred.head()
  1. Convert the DATE column to datetime and sort the DataFrame chronologically.
  2. Filter the data to the years 2010 through 2015 (inclusive).
  3. Create a line chart of UNRATE (the unemployment rate) over DATE for this filtered period, with figsize=(10, 5) and full axis labels and a title.
  4. In which year does unemployment peak within this window? Roughly how many percentage points does it fall by the end of 2015?
Solution 
import pandas as pd
import matplotlib.pyplot as plt

url = "https://raw.githubusercontent.com/isabelhovdahl/skl401/refs/heads/master/data/FRED/FRED_monthly_2010.csv"
fred = pd.read_csv(url)

# 1. Convert and sort
fred["DATE"] = pd.to_datetime(fred["DATE"])
fred = fred.sort_values("DATE")

# 2. Filter to 2010-2015
fred_subset = fred[(fred["DATE"] >= "2010-01-01") & (fred["DATE"] <= "2015-12-01")]
# 3. Plot
fig, ax = plt.subplots(figsize=(10, 5))

ax.plot(fred_subset["DATE"], fred_subset["UNRATE"])

ax.set_xlabel("Year")
ax.set_ylabel("Unemployment rate (%)")
ax.set_title("US unemployment rate, 2010–2015")

plt.tight_layout()
plt.show()
  1. Unemployment peaks in early 2010, close to 10%, reflecting the depth of the recovery still underway after the 2008 financial crisis. By the end of 2015 it has fallen to around 5% — a decline of roughly 4 to 5 percentage points over the six-year window, illustrating the long, gradual nature of labour market recovery after a major recession.
