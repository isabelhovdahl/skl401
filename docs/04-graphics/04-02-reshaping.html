Reshaping data

We cover two reshaping operations — pivot() to convert long data to wide format, and melt() to go back the other way — since the shape of a DataFrame directly affects how easy it is to plot.

import pandas as pd

# ── The gapminder dataset ────────────────────────────────────

url       = "https://raw.githubusercontent.com/plotly/datasets/master/gapminderDataFiveYear.csv"
gapminder = pd.read_csv(url)

gapminder.shape
gapminder.head()
gapminder.info()

gapminder["continent"].unique()
gapminder["country"].nunique()
gapminder["year"].unique()

gapminder.to_csv("data/gapminder.csv", index=False)
print("Saved to data/gapminder.csv")

# ── Wide vs long format ──────────────────────────────────────

# Select 4 countries and 4 years
countries = ["Norway", "Brazil", "Nigeria", "China"]
years     = [1952, 1972, 1992, 2007]

subset = gapminder[
    (gapminder["country"].isin(countries)) &
    (gapminder["year"].isin(years))
]

subset

# ── Converting long to wide with .pivot() ────────────────────

# pivot() reshapes long data into wide format
#   index   = column whose values become row labels
#   columns = column whose values become new column headers
#   values  = column whose values fill the cells
wide = subset.pivot(index="country", columns="year", values="lifeExp").reset_index()

wide

# ── Converting wide to long with .melt() ────────────────────

# melt() reshapes wide data into long format
#   id_vars    = columns to keep as-is
#   var_name   = name for the new column of old column headers
#   value_name = name for the new column of values
long = wide.melt(id_vars = "country", var_name   = "year", value_name = "lifeExp")

long

Exercises

Exercise 4.2.1

For each of the three small DataFrames below, decide whether it is in wide or long format, and explain your reasoning in one sentence.

import pandas as pd

df_a = pd.DataFrame({
    "country": ["Norway", "Norway", "Sweden", "Sweden"],
    "year":    [2000, 2010, 2000, 2010],
    "gdp":     [36000, 60000, 34000, 52000]
})

df_b = pd.DataFrame({
    "country": ["Norway", "Sweden"],
    "gdp_2000": [36000, 34000],
    "gdp_2010": [60000, 52000]
})

df_c = pd.DataFrame({
    "student": ["Alice", "Alice", "Bob", "Bob"],
    "subject": ["Math", "English", "Math", "English"],
    "score":   [88, 76, 91, 82]
})
Solution

df_a is long — each row is a single country-year observation, and year appears as values within a column rather than as separate columns.

df_b is wide — each row is a single country, and each year has its own column (gdp_2000, gdp_2010). The same information as df_a, but reshaped.

df_c is long — each row is a single student-subject observation. Reshaping this to wide would produce one row per student, with Math and English as separate columns.

The general test: in long format, a variable like year or subject is stored as values inside a column. In wide format, those same values become the column headers themselves.
