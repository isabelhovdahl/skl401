Selecting columns and rows

We cover the core tools for subsetting a DataFrame: selecting single and multiple columns, and selecting rows by label with .loc[] or by position with .iloc[].

import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

# ── Selecting a single column ────────────────────────────────

fares = titanic["fare"]
type(fares)
fares.head()

fares.mean()
fares.max()

titanic["survived"].sum()    # total number of survivors

titanic["pclass"].nunique()
titanic["pclass"].unique()
titanic["pclass"].value_counts()

# ── Selecting multiple columns ───────────────────────────────

subset = titanic[["survived", "pclass", "sex", "age", "fare"]]
subset.head()

# ── Selecting rows with .loc[] — label-based ─────────────────

# Single row
titanic.loc[0]

# A range of rows — .loc[] is inclusive on both ends
titanic.loc[0:4]

# A range of rows and specific columns
titanic.loc[0:4, ["survived", "pclass", "sex", "age"]]

# ── Selecting rows with .iloc[] — position-based ────────────

# Rows at positions 0, 1, 2 (stop=3 excluded), first 4 columns
titanic.iloc[0:3, 0:4]

# Two last rows using negative indexing
titanic.iloc[-2]

Exercise 3.2.1

There are two ways to select a single column from a DataFrame, and they return different types of object. Run the code below and compare the output of each line.

import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

result_a = titanic["age"]
result_b = titanic[["age"]]

print(type(result_a))
print(type(result_b))
  1. What type does each expression return?
  2. What is the practical difference between a Series and a single-column DataFrame?
  3. When selecting multiple columns, which of the two forms must you use, and why?
Solution 
import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

result_a = titanic["age"]
result_b = titanic[["age"]]

print(type(result_a))
print(type(result_b))
<class 'pandas.Series'>
<class 'pandas.DataFrame'>

  1. titanic["age"] returns a Series — a single column with its index on the left. titanic[["age"]] returns a DataFrame — a table with one column.

  2. A Series is a one-dimensional object. A DataFrame is two-dimensional — it has both rows and named columns. Most summary methods (.mean(), .sum()) work on both, but some operations — such as saving to CSV or merging — expect a DataFrame.

  3. When selecting multiple columns you must pass a list of column names inside the square brackets, for example titanic[["age", "fare", "pclass"]]. The outer brackets are the selection operator; the inner brackets create the Python list. This always returns a DataFrame regardless of how many columns you select.

Exercise 3.2.3

.loc[] and .iloc[] look similar but follow different slicing rules, and mixing them up is one of the most common sources of off-by-one errors in pandas.

Run both selections below and compare the results.

import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

selection_a = titanic.loc[0:4]
selection_b = titanic.iloc[0:4]
  1. How many rows does each selection return? Why do they differ?
  2. Use .iloc[] to select the last 3 rows of the dataset without hardcoding the total number of rows.
Solution 
import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

selection_a = titanic.loc[0:4]
selection_b = titanic.iloc[0:4]

print(len(selection_a))
print(len(selection_b))
5
4
# Last 3 rows using negative indexing
print(titanic.iloc[-3:])
     survived  pclass     sex   age  sibsp  parch   fare embarked  class  \
888         0       3  female   NaN      1      2  23.45        S  Third   
889         1       1    male  26.0      0      0  30.00        C  First   
890         0       3    male  32.0      0      0   7.75        Q  Third   

       who  adult_male deck  embark_town alive  alone  
888  woman       False  NaN  Southampton    no  False  
889    man        True    C    Cherbourg   yes   True  
890    man        True  NaN   Queenstown    no   True

  1. titanic.loc[0:4] returns 5 rows (rows 0, 1, 2, 3, and 4). .loc[] is inclusive on both ends, so the stop label 4 is included. titanic.iloc[0:4] returns 4 rows (positions 0, 1, 2, 3). .iloc[] follows standard Python slicing rules where the stop is excluded. When the index is the default integer index, the label and the position are the same number — which makes the difference easy to miss. Always double-check which one you are using.

  2. titanic.iloc[-3:] uses negative indexing to count from the end. Omitting the stop returns everything from position -3 to the end of the DataFrame.

As a rule of thumb: use .loc[] when you know the label, and .iloc[] when you need a specific position or want to count from the end.
