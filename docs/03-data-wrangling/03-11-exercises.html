Topic 3 — Further Exercises

These exercises give you an opportunity to consolidate everything covered in Topic 3 before moving on. Unlike the exercises after each video, many of these combine skills from several videos into a single task. Some exercises also introduce new concepts that extend what you have learned.

The exercises are organised into three levels:

  • 🟢 Warm-up — single-skill exercises to check your understanding of the basics.
  • 🟡 Practice — exercises that combine two or more topics.
  • 🔴 Challenge — multi-step tasks that may require you to combine ideas in ways you have not seen before. Some may require looking things up or experimenting.

Exercise 3.11.1

Use the titanic dataset to answer the following questions using groupby and aggregation. Each question requires a single groupby operation.

import pandas as pd

titanic = pd.read_csv("data/titanic.csv")
  1. Compute the mean and median fare paid by passengers in each embarkation port (embarked). Which port has the highest mean fare? Is the mean or median more representative here, and why?
  2. For each combination of passenger class and sex, compute the survival rate and the number of passengers. Sort the result by survival rate in descending order.
  3. Compute the mean age and mean fare for each passenger class.
Solution 
import pandas as pd

titanic = pd.read_csv("data/titanic.csv")

# 1. Mean and median fare by embarkation port
print(
    titanic.groupby("embarked")["fare"]
    .agg(mean_fare="mean", median_fare="median")
    .round(2)

Task 1 Port C (Cherbourg) has the highest mean fare by a considerable margin. The mean is likely less representative than the median here because fare distributions tend to be right-skewed — a small number of first-class passengers paying very high fares pulls the mean upward.

Exercise 3.11.4

This exercise combines filtering, missing value handling, column creation, and groupby to produce a summary of Titanic survival patterns.

import pandas as pd

titanic = pd.read_csv("data/titanic.csv")
  1. Drop rows where embarked is missing. Then fill missing values in age with the median age within each passenger class using groupby and transform. Verify that no missing values remain in either column.
  2. Create a column age_group with the following categories:
    • "child" if age is below 18
    • "adult" if age is between 18 and 59
    • "senior" if age is 60 or above
  3. Compute the survival rate for each combination of age_group and pclass. Sort by survival rate in descending order. Which group had the highest survival rate?
  4. Compute the share of each age_group within each passenger class. Does the age distribution differ meaningfully across classes?
Hint

For step 2, you could write a function with an if/elif/else chain and use .apply(). But pandas also has a vectorised function called np.select() that handles exactly this kind of multi-branch classification without needing a custom function. It takes a list of conditions and a matching list of values:

conditions = [condition1, condition2]
choices    = [value1, value2]
np.select(conditions, choices, default=default_value)

You can look up the documentation for np.select() by running help(np.select) in the console or searching online.

Exercise 3.11.5

This exercise asks you to implement the percentage change calculation two ways: first with a manual loop, and then with a pandas built-in function. The goal is to make a habit of checking whether pandas already has a function for what you want to do, before writing your own.

import pandas as pd

flights = pd.read_csv("data/flights.csv")
flights["date"] = pd.to_datetime(
    flights["year"].astype(str) + "-" + flights["month"],
    format="%Y-%b"
)
flights = flights.sort_values("date").reset_index(drop=True)

Part 1. Without using any pandas function designed for this purpose, compute the month-on-month percentage change in passengers using a for loop. For each row (starting from the second row), use .iloc[] to access the current and previous row’s passenger count, compute the percentage change using the formula below, and store the result in a new column pct_change_manual. The first row should be NaN since there is no previous value to compare against.

\[\text{percentage change} = \frac{\text{current} - \text{previous}}{\text{previous}} \times 100\]

Part 2. pandas has a built-in method that computes percentage change directly on a Series in a single line. Look up the documentation for pd.Series.pct_change() and use it to create a second column pct_change_pandas. Multiply by 100 to match the scale of Part 1.

Part 3. Drop rows where the percentage change is missing, and verify that the two columns are identical. Then filter to rows where the month-on-month change exceeds 20% and print those rows. In which years and months does passenger growth spike most strongly?

Hint

For Part 1, initialize the new column with NaN for every row, for example using flights["pct_change_manual"] = None. Then loop over the row positions with for i in range(1, len(flights)), and use .iloc[] to access the current row (flights.iloc[i]) and the previous row (flights.iloc[i - 1]). Assign the computed value back into the column at position i using flights.loc[i, "pct_change_manual"] = ....

Solution 
import pandas as pd

flights = pd.read_csv("data/flights.csv")
flights["date"] = pd.to_datetime(
    flights["year"].astype(str) + "-" + flights["month"],
    format="%Y-%b"
)
flights = flights.sort_values("date").reset_index(drop=True)

# Part 1: manual loop
flights["pct_change_manual"] = None

for i in range(1, len(flights)):
    current  = flights.iloc[i]["passengers"]
    previous = flights.iloc[i - 1]["passengers"]
    flights.loc[i, "pct_change_manual"] = (current - previous) / previous * 100

flights["pct_change_manual"] = flights["pct_change_manual"].astype(float)

# Part 2: pandas built-in
flights["pct_change_pandas"] = flights["passengers"].pct_change() * 100

The two columns are identical. A short note on the loop mechanics: initializing the column with flights["pct_change_manual"] = None creates an object-dtype column, since None is not numeric. Assigning floats into individual cells with .loc[] does not automatically upgrade the column’s dtype, so .astype(float) is needed afterward to convert it to a proper numeric column — this also turns the remaining None in the first row into a genuine NaN. Even after this conversion, comparing the two columns with == will mark the first row as unequal, because NaN == NaN always evaluates to False in pandas, regardless of how the missing value was created. Alternatively you can use .equals() rather than == as this treats matching NaN positions as equals:

flights["pct_change_manual"].round(6).equals(flights["pct_change_pandas"].round(6))

The broader lesson: a for loop with .iloc[] works, but it is slow, verbose, and easy to get wrong — for example by forgetting to handle the first row, or making an off-by-one error in the row indices. Before reaching for a loop, it is always worth checking whether pandas already has a built-in vectorised function for the operation. .pct_change(), .cumsum(), .diff(), and .shift() cover many common row-wise operations without needing a loop or a custom function at all.

Exercise 3.11.7

This exercise combines np.where() and np.select() with groupby to classify and compare countries in the healthexp dataset.

import pandas as pd
import numpy as np

healthexp = pd.read_csv("data/healthexp.csv")
healthexp.columns = [col.lower().replace(" ", "_") for col in healthexp.columns]
  1. Use np.where() to create a column high_life_exp that is 1 if life expectancy is above the overall median and 0 otherwise.
  2. Use np.select() to create a column spending_tier with three categories:
    • "low" if spending is below the 33rd percentile
    • "medium" if spending is between the 33rd and 66th percentile
    • "high" if spending is above the 66th percentile
  3. Compute the mean life expectancy for each spending_tier. Does higher spending reliably predict higher life expectancy across all tiers?

Task 5 Upsampling with .asfreq() creates a row for every single day between the first and last date in the index, but only the original month-start dates have a value — every other day is NaN. This is the gap-filling behavior in action: resample() builds its date range from the calendar, not from the rows that happen to exist, so it makes the resolution mismatch between the original data and the daily target explicit rather than silently ignoring it. groupby() could not do this at all, since there is nothing in the original data to group by day.

Task 6 .ffill() propagates each month’s total forward to every day until the next known value, so every day in a given month temporarily holds that month’s total. Dividing by .dt.days_in_month then converts each day’s value from “the whole month’s total” to “this day’s share of the total,” correctly accounting for the fact that months have different lengths (28–31 days). The result is a daily series whose values sum back up to the original monthly totals — a reasonable estimate given the assumption of even distribution, even though the true day-to-day pattern (e.g. weekday vs weekend travel) is not recoverable from monthly data alone.

import pandas as pd

base_url = "https://raw.githubusercontent.com/isabelhovdahl/skl401/refs/heads/master/03-data-wrangling/data/FRED/"
decades = [1950, 1960, 1970, 1980, 1990, 2000, 2010]

# 1. Import each decade's file in a loop
fred_dfs = []

for decade in decades:
    url = base_url + f"FRED_monthly_{decade}.csv"
    df = pd.read_csv(url)
