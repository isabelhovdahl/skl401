Importing data

We introduce the most common ways to get data into Python — loading built-in datasets via seaborn, and importing CSV and Excel files — and cover how to save DataFrames back to files.

import pandas as pd
import seaborn as sns

# ── Loading a built-in dataset with seaborn ──────────────────

titanic = sns.load_dataset("titanic")
titanic.shape
titanic.head()

# Important checks
# titanic.info()
# titanic.describe()

# ── Saving data to a file ────────────────────────────────────

# Save to CSV
titanic.to_csv("data/titanic.csv", index=False)
print("Saved to data/titanic.csv")

# Save to Excel
titanic.to_excel("data/titanic.xlsx", index=False, sheet_name="titanic")
print("Saved to data/titanic.xlsx")

# ── Loading from a file ──────────────────────────────────────

titanic_from_csv = pd.read_csv("data/titanic.csv")
titanic_from_csv.head()

titanic_from_excel = pd.read_excel("data/titanic.xlsx", sheet_name="titanic")
titanic_from_excel.head()

# ── The sep parameter for European CSV files ─────────────────

# Save a semicolon-separated version to demonstrate
titanic.to_csv("data/titanic_semicolon.csv", index=False, sep=";")

# Load it back — must specify sep=";"
titanic_semi = pd.read_csv("data/titanic_semicolon.csv", sep =";")
titanic_semi.head()

# ── Saving and loading the other datasets ────────────────────

healthexp = sns.load_dataset("healthexp")
flights   = sns.load_dataset("flights")

healthexp.to_csv("data/healthexp.csv", index=False)
flights.to_csv("data/flights.csv", index=False)

print("healthexp saved:", healthexp.shape)
print("flights saved:",   flights.shape)

Exercise 3.1.3

This exercise explores the difference between relative and absolute paths, and what can go wrong when you use the wrong one.

Run the following code and look at the path that is printed:

import os
print(os.getcwd())

os.getcwd() returns the working directory — the folder Python is currently treating as its starting point. When you use a relative path like "data/titanic.csv", Python looks for a data/ folder inside this location.

Now answer the following questions:

  1. Does the path printed match the project folder you set up at the start of the course — the folder that contains your scripts and your data/ subfolder?

  2. Load titanic.csv using the full absolute path you found in Exercise 3.1.2 instead of the relative path. Confirm the result is identical. Note: if you are on Windows, your path will use backslashes (e.g. C:\Users\yourname\...). Python requires forward slashes in paths, so replace any backslashes with forward slashes before running the code.

  3. Suppose a classmate copies only the script file to their Desktop and runs it from there, without copying the data/ folder. Which path would work for them — the relative path or the absolute path you used in step 2? Explain why.

Solution 
import os
print(os.getcwd())
# Replace this path with the absolute path on your own computer
titanic_abs = pd.read_csv("/Users/yourname/Documents/skl401/data/titanic.csv")
titanic_abs.shape

  1. Yes — if you opened Positron from your project folder and are running scripts from there, os.getcwd() should return that folder. If it shows something unexpected, check that Positron has the right folder open (Open → Open Folder…).

  2. The result is identical — read_csv() does not care whether you provide a relative or absolute path, as long as the path points to the same file.

  3. Neither path would work as-is. The relative path "data/titanic.csv" would fail because there is no data/ folder on the Desktop. The absolute path would also fail because it points to a location on your computer, not theirs. To share a script that others can run, you either need to share the whole project folder (scripts and data together), or load data from a URL. Hard-coding absolute paths into scripts is a common mistake — they silently break the moment anyone else tries to run the code.

Exercise 3.1.4

The code below saves a small DataFrame to CSV and loads it back in. Run it and inspect the result.

import pandas as pd

products = pd.DataFrame({
    "product":  ["coffee", "tea", "juice"],
    "price":    [29.90, 35.00, 12.50],
    "quantity": [3, 5, 0]
})

products.to_csv("data/products.csv")

products_loaded = pd.read_csv("data/products.csv")
print(products_loaded)
  1. How many columns does products_loaded have compared to products? What is the extra column called, and where did it come from?
  2. Fix the to_csv() call so that the extra column does not appear when you reload the file.
Solution
  1. products_loaded has 4 columns instead of 3. The extra column is called Unnamed: 0 and contains the numbers 0, 1, 2. It comes from the row index — when you save a DataFrame without specifying index=False, pandas writes the index as the first column in the file. When you reload the file, pandas reads that column as ordinary data because it has no way of knowing it was the index.
import pandas as pd

products = pd.DataFrame({
    "product":  ["coffee", "tea", "juice"],
    "price":    [29.90, 35.00, 12.50],
    "quantity": [3, 5, 0]
})

products.to_csv("data/products.csv")

products_loaded = pd.read_csv("data/products.csv")
print(products_loaded)
   Unnamed: 0 product  price  quantity
0           0  coffee   29.9         3
1           1     tea   35.0         5
2           2   juice   12.5         0
  1. Add index=False to the to_csv() call:
import pandas as pd

products = pd.DataFrame({
    "product":  ["coffee", "tea", "juice"],
    "price":    [29.90, 35.00, 12.50],
    "quantity": [3, 5, 0]
})

products.to_csv("data/products.csv", index=False)

products_loaded = pd.read_csv("data/products.csv")
print(products_loaded)
  product  price  quantity
0  coffee   29.9         3
1     tea   35.0         5
2   juice   12.5         0

This is one of the most common mistakes when saving DataFrames. Always use index=False unless you have specifically set a meaningful index (such as a date or a country name) that you want to preserve.

Exercise 3.1.5

A colleague sends you a CSV file exported from Excel. When you try to load it, something looks wrong:

import pandas as pd

df = pd.read_csv("data/titanic_semicolon.csv")
print(df.shape)
print(df.head())

Instead of 891 rows and 15 columns, the output shows 891 rows and 1 column, and the single column contains values like "survived;pclass;sex;age;fare;...".

  1. What has gone wrong?
  2. Fix the read_csv() call so the data loads correctly with all 15 columns.

Note: the file titanic_semicolon.csv was created by the video code. If you do not have it yet, run this first:

titanic = pd.read_csv("data/titanic.csv")
titanic.to_csv("data/titanic_semicolon.csv", index=False, sep=";")
Solution

  1. The file uses semicolons as the separator between columns instead of commas. By default, read_csv() expects commas. When it finds none, it treats each entire row as a single value and returns a DataFrame with one column containing all the fields joined together.

  2. Add sep=";" to tell pandas to split on semicolons instead:

import pandas as pd

df = pd.read_csv("data/titanic_semicolon.csv", sep=";")
print(df.shape)
print(df.head())
(891, 15)

Semicolon-separated files are common in Europe, where the comma is used as the decimal separator in numbers. If you ever load a CSV and get a single column with all the data crammed together, a mismatched separator is almost always the cause. Open the raw file in a text editor to see which separator it actually uses.
