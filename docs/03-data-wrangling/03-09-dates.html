Working with dates

We cover how to convert string columns to proper datetime objects, how to extract date components such as year or month using the .dt accessor, and how to filter and group data by date.

import pandas as pd

# ── Converting strings to datetime ───────────────────────────

# A small DataFrame with dates stored as strings
df = pd.DataFrame({
    "date":    ["2020-01-15", "2019-06-30", "2021-11-03", "2018-03-22"],
    "country": ["Norway", "Germany", "France", "Japan"],
    "value":   [100, 200, 150, 180],
})
df
df.dtypes

# Convert to datetime
df["date"] = pd.to_datetime(df["date"])
df
df.dtypes

# Specifying the format explicitly
# (%Y is four-digit year, %m is month (01-12), %d is day of the month (01-31))
df["date"] = pd.to_datetime(df["date"], format="%Y-%m-%d")
df

# ── Applying this to the flights dataset ─────────────────────

flights = pd.read_csv("data/flights.csv")
flights.head()
flights.dtypes

# Build a date column from year and month
flights["date"] = flights["year"].astype(str) + "-" + flights["month"]

# %Y is four-digit year, %b is an abbreviated month name like "Jan"
flights["date"] = pd.to_datetime(flights["date"], format="%Y-%b")

flights.head()

# ── Extracting date components with .dt ──────────────────────

flights["year_dt"]   = flights["date"].dt.year
flights["month_num"] = flights["date"].dt.month
flights["quarter"]   = flights["date"].dt.quarter

flights.head()

# ── Filtering by date range ──────────────────────────────────

# Flights in 1955 and 1956
mid_50s = flights[
    (flights["date"] >= "1955-01-01") &
    (flights["date"] <= "1956-12-31")
]
mid_50s.shape
mid_50s

# ── Grouping by date components ──────────────────────────────

# Total annual passengers
annual = flights.groupby(flights["date"].dt.year)["passengers"].sum().reset_index()
annual

# Rename columns
annual = annual.rename(columns={"date": "year", "passengers": "total_passengers"})
annual.head()

Exercise 3.9.1

The DataFrame below contains sales records with dates stored as strings.

import pandas as pd

sales = pd.DataFrame({
    "date":    ["2020-03-15", "2019-11-02", "2021-07-28", "2018-05-10",
                "2020-01-22", "2019-08-14", "2021-03-05", "2018-12-30"],
    "product": ["Coffee", "Tea", "Juice", "Coffee", "Tea", "Juice",
                "Coffee", "Tea"],
    "revenue": [1200, 850, 640, 1100, 920, 710, 1350, 780]
})
  1. Sort the DataFrame by date and print the result. Does the sort order look correct? What is happening, and why?
  2. Filter the DataFrame to sales from 2020 onwards using sales["date"] >= "2020-01-01". Does the result look correct?
  3. Convert the date column to datetime using pd.to_datetime() and check the dtype before and after. Then repeat the sort and filter from steps 1 and 2. How do the results differ?

  1. The sort order looks correct in this case because the dates are in YYYY-MM-DD format, which happens to sort correctly as a string because the most significant component (year) comes first.   3. After converting to datetime, both the sort and the filter work correctly and for the right reasons. The dtype changes from str to datetime64[ns], confirming that pandas now treats the column as a proper point in time rather than a sequence of characters.

Exercise 3.9.2

The DataFrame below contains patient visit records with dates in DD/MM/YYYY format — common in European data exports.

import pandas as pd

visits = pd.DataFrame({
    "date":       ["02/01/2021", "06/05/2020", "11/03/2019",
                   "04/08/2022", "01/09/2021", "09/07/2020"],
    "patient_id": [101, 102, 103, 104, 105, 106],
    "diagnosis":  ["flu", "checkup", "injury", "flu", "checkup", "injury"]
})
  1. Convert the date column to datetime without specifying a format= argument and print the result. The conversion succeeds without any error — but do the dates look correct? Compare the converted dates against the original strings and explain what pandas has done wrong.
  2. Fix the conversion by specifying the correct format= argument for DD/MM/YYYY dates. Print the result and verify it now matches the original strings.
  3. Add two new columns to visits: year and month_num, extracted from the correctly converted date column using the .dt accessor.
Hint

The format codes you need is "%d/%m/%Y" — %d for day, %m for month number, and %Y for four-digit year. Always verify the converted dates look sensible after conversion — ambiguous formats can silently produce wrong results with no error or warning.

Solution 
import pandas as pd

visits = pd.DataFrame({
    "date":       ["02/01/2021", "06/05/2020", "11/03/2019",
                   "04/08/2022", "01/09/2021", "09/07/2020"],
    "patient_id": [101, 102, 103, 104, 105, 106],
    "diagnosis":  ["flu", "checkup", "injury", "flu", "checkup", "injury"]
})

# 1. Convert without format
visits["date_wrong"] = pd.to_datetime(visits["date"])
print(visits[["date", "date_wrong", "patient_id"]])
         date date_wrong  patient_id
0  02/01/2021 2021-02-01         101
1  06/05/2020 2020-06-05         102
2  11/03/2019 2019-11-03         103
3  04/08/2022 2022-04-08         104
4  01/09/2021 2021-01-09         105
5  09/07/2020 2020-09-07         106
# 2. Convert with correct format
visits["date"] = pd.to_datetime(visits["date"], format="%d/%m/%Y")
print(visits[["date", "date_wrong", "patient_id"]])
        date date_wrong  patient_id
0 2021-01-02 2021-02-01         101
1 2020-05-06 2020-06-05         102
2 2019-03-11 2019-11-03         103
3 2022-08-04 2022-04-08         104
4 2021-09-01 2021-01-09         105
5 2020-07-09 2020-09-07         106
# 3. Extract year and month
visits["year"]      = visits["date"].dt.year
visits["month_num"] = visits["date"].dt.month
print(visits.head())
        date  patient_id diagnosis date_wrong  year  month_num
0 2021-01-02         101       flu 2021-02-01  2021          1
1 2020-05-06         102   checkup 2020-06-05  2020          5
2 2019-03-11         103    injury 2019-11-03  2019          3
3 2022-08-04         104       flu 2022-04-08  2022          8
4 2021-09-01         105   checkup 2021-01-09  2021          9

  1. The conversion succeeds but the dates are wrong. pandas assumes the American MM/DD/YYYY format by default and interprets the first number as the month and the second as the day — so "02/01/2021" becomes February 1st instead of January 2nd. Because all day values in this dataset happen to be 12 or below, every date is a valid calendar date in both interpretations, so pandas raises no error. This is one of the most dangerous silent bugs when working with dates: the code runs cleanly and the output looks plausible, but every date is wrong. Always cross-check a few converted values against the original strings to catch this.

  2. With format="%d/%m/%Y", pandas correctly reads the first number as day and the second as month, producing the intended dates.

Exercise 3.9.3

The DataFrame below records daily weather observations with the date split across three separate columns.

import pandas as pd

weather = pd.DataFrame({
    "year":      [2021, 2021, 2021, 2022, 2022, 2022],
    "month":     [3, 6, 12, 1, 7, 11],
    "day":       [15, 22, 1, 8, 30, 14],
    "city":      ["Oslo", "Oslo", "Oslo", "Oslo", "Oslo", "Oslo"],
    "temp_c":    [2.1, 18.4, -3.2, -5.1, 22.8, 4.6],
    "rainfall":  [5.2, 0.0, 8.1, 2.3, 12.4, 6.7]
})
  1. Construct a date column by combining year, month, and day into a single string in YYYY-MM-DD format. Remember that month and day are integers and will need to be converted to strings first.
  2. Convert the date column to datetime using the appropriate format= argument and verify the dtype.
  3. Add a days_since_start column that computes the number of days between each observation and the earliest date in the dataset. Print the result.
Hint

The difference between two datetime values produces a timedelta — an object representing a duration rather than a point in time. You can extract the number of days from a timedelta using .dt.days.

Solution 
import pandas as pd

weather = pd.DataFrame({
    "year":      [2021, 2021, 2021, 2022, 2022, 2022],
    "month":     [3, 6, 12, 1, 7, 11],
    "day":       [15, 22, 1, 8, 30, 14],
    "city":      ["Oslo", "Oslo", "Oslo", "Oslo", "Oslo", "Oslo"],
    "temp_c":    [2.1, 18.4, -3.2, -5.1, 22.8, 4.6],
    "rainfall":  [5.2, 0.0, 8.1, 2.3, 12.4, 6.7]
})

# 1. Build date string
weather["date"] = (
    weather["year"].astype(str) + "-" +
    weather["month"].astype(str) + "-" +
    weather["day"].astype(str)
)
print(weather["date"].head())
0    2021-3-15
1    2021-6-22
2    2021-12-1
3     2022-1-8
4    2022-7-30
Name: date, dtype: str
# 2. Convert to datetime
weather["date"] = pd.to_datetime(weather["date"], format="%Y-%m-%d")
print(weather["date"].dtype)
datetime64[us]
# 3. Days since earliest date
weather["days_since_start"] = (
    weather["date"] - weather["date"].min()
).dt.days
print(weather[["date", "temp_c", "days_since_start"]])
        date  temp_c  days_since_start
0 2021-03-15     2.1                 0
1 2021-06-22    18.4                99
2 2021-12-01    -3.2               261
3 2022-01-08    -5.1               299
4 2022-07-30    22.8               502
5 2022-11-14     4.6               609

Subtracting two datetime columns produces a timedelta — a duration. Calling .dt.days extracts the integer number of days from that duration. This pattern is useful whenever you need to compute elapsed time: days since first purchase, days between diagnosis and treatment, days until a deadline, and so on.

Exercise 3.9.5

Use the flights dataset with its date column to practice filtering by date range and comparing periods.

import pandas as pd

flights = pd.read_csv("data/flights.csv")
flights["date"] = pd.to_datetime(
    flights["year"].astype(str) + "-" + flights["month"],
    format="%Y-%b"

  1. Filter flights to the period from January 1949 to December 1951 inclusive. Assign the result to early and print its shape.
  2. Filter flights to the period from January 1958 to December 1960 inclusive. Assign the result to late and print its shape.
  3. Compute the mean monthly passengers for each period and the percentage increase from the early to the late period. What does this tell you about the growth of air travel over this decade?
  4. Within the late period, filter further to summer months only (July, August, September) and compute the mean monthly passengers. How does this compare to the overall mean for the late period?
Solution 
import pandas as pd

flights = pd.read_csv("data/flights.csv")
flights["date"] = pd.to_datetime(
    flights["year"].astype(str) + "-" + flights["month"],
    format="%Y-%b"
)

# 1. Early period
early = flights[
    (flights["date"] >= "1949-01-01") &
    (flights["date"] <= "1951-12-31")
]
print(early.shape)
(36, 4)
# 2. Late period
late = flights[
    (flights["date"] >= "1958-01-01") &
    (flights["date"] <= "1960-12-31")
]
print(late.shape)
(36, 4)
# 3. Mean monthly passengers and percentage increase
early_mean = early["passengers"].mean()
late_mean  = late["passengers"].mean()
pct_change = (late_mean - early_mean) / early_mean * 100

print(f"Early mean: {early_mean:.1f}")
print(f"Late mean:  {late_mean:.1f}")
print(f"Increase:   {pct_change:.1f}%")
Early mean: 145.5
Late mean:  428.5

# 4. Summer months in the late period
summer_late = late[late["date"].dt.month.isin([7, 8, 9])]
print(f"Summer mean: {summer_late['passengers'].mean():.1f}")
print(f"Overall mean: {late['passengers'].mean():.1f}")
Summer mean: 522.9
Overall mean: 428.5

  1. Mean monthly passengers more than doubled from the early to the late period — reflecting the rapid expansion of commercial aviation in the 1950s driven by the introduction of jet aircraft and rising incomes.

  2. Summer passengers in the late period are considerably higher than the overall late-period mean, confirming that the seasonal peak in Q3 identified in exercise 3.9.4 holds throughout the decade and becomes more pronounced as overall travel volumes grow.
